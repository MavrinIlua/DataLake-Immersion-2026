Архитектура данных

🏞️ Озеро данных (Data Lake)

Хранилище неструктурированных данных в сыром виде. Как настоящее озеро: туда стекаются реки данных (логи, фото, видео, текст) без предварительной очистки.

Для чего: Глубокая аналитика, машинное обучение, хранение истории.

🏢 Хранилище данных (Data Warehouse)

Структурированное хранилище очищенных данных. Как библиотека с каталогом: всё разложено по полочкам и готово к выдаче.

Для чего: Бизнес-отчетность, дашборды, быстрый SQL-анализ.

🏠 Озерохранилище (Lakehouse)

Современный гибрид. Объединяет гибкость озера и надежность хранилища. Позволяет делать ML и отчеты на одних данных.

Технологии: Delta Lake, Apache Iceberg.

Управление данными: Как избежать хаоса

Создание озера данных — это не просто копирование файлов. Без правильного управления оно превращается в "болото данных" (Data Swamp), где невозможно ничего найти.

Проблема Решение
Непонятно, чьи это данные Data Governance (управление доступом и метаданными)
Данные устарели или дублируются Data Quality (автоматические проверки качества)
Нельзя найти нужный файл Data Catalog (каталогизация и поиск)
